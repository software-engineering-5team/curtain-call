# 폴더 역할

`harness/` 폴더는 Curtain Call 서비스의 **AI 작업 안전망(Engineering Harness)** 이다.
AI가 코드를 생성·수정·리뷰할 때 반드시 참조해야 할 정책(rules), 검증(tests), 프롬프트(prompts)를 한 곳에 모아 둔다.
구현은 `app/`, `components/`, `lib/` 에 있고, 그 구현이 따라야 할 **불변 규칙**은 본 폴더에 있다.

# 포함 파일

- `rules/` — 도메인 제약 조건을 기술한 JSON 파일 (사람과 AI 모두 읽을 수 있는 정책 카탈로그)
- `tests/` — 핵심 비즈니스 로직을 검증하는 TypeScript 테스트 코드
- `prompts/` — AI에게 작업 지시를 줄 때 재사용 가능한 표준 프롬프트(Markdown)
- 각 하위 폴더에는 별도의 `CLAUDE.md` 가 존재하며 우선해서 읽는다.

# 작업 규칙

1. AI는 코드를 작성/수정하기 전에 **반드시 `harness/rules/*.json` 과 `AGENTS.md` 를 먼저 로드**한다.
2. 새 비즈니스 규칙이 생기면 `rules/` 의 적절한 JSON에 항목을 추가하고 `tests/` 에 케이스를 함께 추가한다.
3. 규칙은 코드보다 항상 더 엄격하게 정의한다. 코드가 규칙을 어기면 코드가 잘못된 것이다.
4. `prompts/` 의 템플릿을 수정할 때는 변경 사유를 PR 설명에 남긴다.
5. `harness/` 내부 파일에는 UI 코드, 환경별 비밀값, 사용자 데이터를 절대 두지 않는다.

# 관련 폴더

- `/AGENTS.md` — 서비스 전체 규칙. 본 폴더의 상위 정책.
- `/todo/` — 본 폴더 규칙을 따르는 실제 작업 단위들.
- `/app/`, `/components/`, `/lib/` — 본 폴더 규칙이 적용되는 실제 구현.

# 완료 체크리스트

- [ ] `rules/`, `tests/`, `prompts/` 세 하위 폴더가 모두 존재하고 각각 `CLAUDE.md` 를 갖고 있다.
- [ ] 새로 추가된 도메인 규칙은 JSON과 테스트 양쪽에 모두 반영되었다.
- [ ] AGENTS.md 의 인증/예약/좌석/취소 규칙과 충돌하지 않는다.
- [ ] AI가 prompts 템플릿을 사용했을 때 일관된 결과가 나오도록 예시가 갱신되었다.
