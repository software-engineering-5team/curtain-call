# 폴더 역할

AI(Claude 등)가 기능 구현, 테스트 작성, 에러 처리, 코드 리뷰를 일관된 품질로 수행하도록 돕는 **재사용 가능한 프롬프트 템플릿** 모음이다.
사람이 매번 같은 컨텍스트를 다시 입력하지 않도록, 도메인 컨텍스트 + 작업 지시 + 산출물 형식이 한 파일에 패키징된다.

# 포함 파일

- `api-generation.md` — API 라우트/핸들러 생성 프롬프트
- `test-generation.md` — `harness/tests/*` 테스트 작성 프롬프트
- `error-handling.md` — 도메인 에러 코드와 사용자 메시지 작성 프롬프트
- `business-logic.md` — `lib/`, `app/` 의 비즈니스 로직 구현 프롬프트
- `review-checklist.md` — PR 리뷰 시 사용할 체크리스트(Self-review/Bot-review 공용)

# 작업 규칙

1. 모든 프롬프트는 `AGENTS.md` 와 `harness/rules/*.json` 을 컨텍스트로 명시 로드한 뒤 작업하도록 지시한다.
2. 출력 형식(코드 블록, 파일명, JSON 스키마 등)을 항상 명시한다.
3. "이 폴더의 규칙과 충돌하면 작업을 멈추고 사용자에게 묻는다" 라는 안전장치를 포함한다.
4. 프롬프트는 한국어로 작성한다(서비스 톤 유지).
5. 새 프롬프트를 추가할 때 본 CLAUDE.md 의 "포함 파일" 항목도 함께 갱신한다.

# 관련 폴더

- `/AGENTS.md` — 프롬프트가 항상 참조해야 하는 정책
- `/harness/rules/` — 규칙 카탈로그
- `/harness/tests/` — `test-generation.md` 의 산출물이 위치하는 곳
- `/todo/` — 프롬프트로 자동화된 작업이 추적되는 곳

# 완료 체크리스트

- [ ] 5개 프롬프트 파일이 모두 존재한다.
- [ ] 각 프롬프트가 AGENTS.md / rules JSON 경로를 명시 참조한다.
- [ ] 프롬프트 출력의 파일 위치(`harness/tests/...`, `app/...` 등)가 명확히 지정되어 있다.
- [ ] 프롬프트가 도메인 용어(예매/대여/좌석)를 일관되게 사용한다.
