# 폴더 역할

`harness/rules/*.json` 에 선언된 도메인 규칙이 실제 코드에서 깨지지 않는지 검증하는 **테스트 하네스(Testing Harness)** 이다.
UI/렌더링 테스트가 아니라, 비즈니스 로직과 불변식(invariant)을 검증하는 데 집중한다.

# 포함 파일

- `reservation.test.ts` — 좌석 예매 성공/실패 시나리오
- `seat-booking.test.ts` — 좌석 상태 머신, 중복 예매 방지
- `concurrency.test.ts` — 동시 요청 처리(레이스 컨디션 방어)
- `cancellation.test.ts` — 대여/예매 취소 정책
- `validation.test.ts` — 필수값/형식/시간 검증

테스트 러너는 Node 표준 `node:test` 를 가정한다. 별도 라이브러리 없이 실행 가능해야 한다.
```bash
npx tsx --test harness/tests/*.test.ts
```

# 작업 규칙

1. 각 테스트 파일은 한 도메인의 규칙만 다룬다. 도메인 간 결합 테스트는 별도 파일로 분리한다.
2. 모든 테스트는 **규칙 코드(예: RES-003)** 를 주석/타이틀에 포함한다. 코드를 통해 규칙으로 역추적 가능해야 한다.
3. 테스트는 결정적(deterministic)이어야 한다. 시간 의존 테스트는 시간을 주입(inject)한다.
4. 외부 네트워크/DB 의존성을 두지 않는다. 모든 데이터는 fixture 또는 mock 으로 구성한다.
5. 새 규칙 JSON이 추가되면 **같은 PR** 에서 대응 테스트를 추가한다.

# 관련 폴더

- `/harness/rules/` — 검증 대상 규칙의 출처
- `/lib/` — 비즈니스 로직 구현체(예: 충돌 감지, 좌석 점유)
- `/AGENTS.md` — 정책의 최상위 출처
